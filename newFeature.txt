Git Pull і Git Fetch — у чому різниця?
Git Fetch і Git Pull — це команди для отримання змін із віддаленого репозиторію. Вони схожі, але працюють трохи по-різному.

Git Fetch — "Просто перевір, що змінилося"
Коли ти виконуєш git fetch, Git перевіряє, чи є нові зміни у віддаленому репозиторії, і завантажує їх, але не зливає з твоїм поточним кодом.

Що це означає:

Ти отримуєш нові коміти у свою локальну копію.
Твій робочий каталог залишається таким, як був.
Можна переглянути зміни і вирішити, що з ними робити.
Приклад:
git fetch origin main
Що станеться:
Git завантажить зміни з гілки main, але твій код не зміниться, поки ти не виконаєш злиття (merge).

Git Pull — "Завантаж і одразу злий"
git pull — це комбінація команд git fetch + git merge. Він завантажує зміни і одразу зливає їх з твоїм поточним кодом.

Що це означає:

Ти отримуєш оновлення і одразу бачиш їх у своєму робочому каталозі.
Якщо є конфлікти, доведеться їх вирішити.
Приклад:
git pull origin main
Що станеться:
Git завантажить і зіллє зміни з main у твою поточну гілку.

Коли використовувати?
git fetch — коли ти хочеш спочатку перевірити зміни, перш ніж зливати їх. Це безпечніше.
git pull — якщо ти впевнений, що хочеш одразу отримати і злитись зі змінами.
Уяви це так:

git fetch — це як переглянути новини на телефоні.
git pull — це одразу завантажити й прочитати всі повідомлення.
