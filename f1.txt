Fast-forward (швидке злиття)
Це найпростіший тип злиття. Він відбувається, коли немає ніяких розбіжностей між гілками, і Git просто "пересуває" основну гілку до точки, де знаходиться друга.

Як це виглядає:
У тебе є гілка main і додаткова гілка feature.
У main нічого не змінилося з моменту створення feature.
Git просто переміщує main вперед, ніби вона завжди була там. 
Ніяких конфліктів чи нових комітів не створюється.

Приклад команди:
git merge feature
Коли використовується:
Якщо зміни тільки в одній гілці, і немає інших комітів у main.

Recursive Merge (Рекурсивне злиття)
Це стандартний алгоритм злиття в Git, який використовується, коли потрібно об’єднати дві гілки, і в кожній з них були окремі зміни в одному й тому ж файлі.

Як це працює:
Git аналізує спільного предка двох гілок, а потім намагається об’єднати зміни з обох сторін.
Якщо обидві гілки змінили один і той самий фрагмент коду, виникає конфлікт.
Приклад:
У гілці main і гілці feature ти змінив один і той самий рядок коду. Git не знає, яку версію залишити, і видає merge conflict.
